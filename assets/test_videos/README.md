# 测试视频说明

你好，这是一些说明！视频大师辛苦了（鞠躬）！

## 目录结构

```
test_videos/
├── fall_detection/      # 跌倒检测测试视频
├── fire_detection/      # 火灾检测测试视频
└── normal_scenarios/    # 正常场景视频（用于对比）
```

## 视频要求

### 文件格式
- 支持格式: `.mp4`, `.avi`, `.mov`, `.mkv`
- 推荐格式: `.mp4` (H.264编码)

### 时长
- 单个测试场景: 10-30秒
- 完整测试视频: 1-3分钟

## 视频采集指南

### 跌倒检测视频 (`fall_detection/`)

**拍摄要点**:
1. ✅ 从监控视角拍摄（俯视角度，高度约2-2.5米）
2. ✅ 包含完整的跌倒过程：站立 → 失去平衡 → 倒地
3. ✅ 人物在画面中占比适中（不要太远或太近）
4. ✅ 光照充足，避免逆光

**场景多样性**:
- 不同姿势的跌倒（向前、向后、侧倒）
- 不同服装（深色、浅色、长袖、短袖）
- 不同地面（瓷砖、木地板、地毯）
- 不同光照条件（白天、夜晚、灯光）

**命名规范**:
```
fall_forward_bright_01.mp4
fall_backward_dim_01.mp4
fall_side_normal_02.mp4
```

### 火灾检测视频 (`fire_detection/`)

**拍摄要点（其实我（JYY）建议到时候直接不拍摄了，用Sora2生成）**:
1. ✅ 拍摄明火或浓烟场景
2. ✅ 确保安全！建议使用蜡烛、打火机等小型可控火源
3. ✅ 包含火焰从小到大的过程
4. ✅ 背景尽量贴近家居环境

**安全提示**:
⚠️ 请在安全可控的环境下拍摄
⚠️ 准备灭火器材
⚠️ 避免在易燃物附近拍摄

**场景示例**:
- 蜡烛火焰
- 厨房炉灶
- 烟雾效果（可用香烟烟雾模拟）

**命名规范**:
```
fire_candle_01.mp4
fire_kitchen_stove_01.mp4
smoke_heavy_01.mp4
```

### 正常场景视频 (`normal_scenarios/`)

**目的**: 用于对比测试，确保系统不会误报

**拍摄要点**:
1. ✅ 日常生活场景：走路、坐下、站起、做家务
2. ✅ 与监控视角保持一致
3. ✅ 包含人物的各种正常动作

**场景示例**:
- 老人正常走路
- 坐在沙发上看电视
- 在厨房做饭
- 打扫卫生

**命名规范**:
```
normal_walking_01.mp4
normal_sitting_01.mp4
normal_cooking_01.mp4
```

## 使用示例

### 运行单个测试视频

```bash
python main.py --source video --path assets/test_videos/fall_detection/fall_forward_bright_01.mp4
```

### 批量测试

```bash
python scripts/batch_test.py --input assets/test_videos/fall_detection/
```

## 视频获取建议

### 自行拍摄
- 使用咱们这段时间要购置的摄像头拍摄
- 找同学配合表演跌倒（注意安全！）
- 使用三脚架固定视角

### 公开数据集（可选）
- **UR Fall Detection Dataset**: 包含多种跌倒场景

### 注意事项
⚠️ 如果使用公开数据集，请遵守其许可协议
⚠️ 不要上传包含他人隐私的视频到公共仓库

## 测试检查清单

测试前请确保：
- [ ] 至少有3个跌倒场景视频
- [ ] 至少有2个正常场景视频（用于对比）
- [ ] 视频清晰度足够（能看清人物动作）
- [ ] 视频格式正确（推荐mp4）
- [ ] 文件命名规范

如果结果不符合预期，需要调整 `config/detection_config.yaml` 中的阈值参数。
